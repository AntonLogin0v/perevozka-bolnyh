# От койки до койки - Одностраничный сайт

Копия одностраничного сайта для перевозки лежачих больных и колясочников.

## Структура проекта

- `index.html` - основная HTML структура сайта
- `styles.css` - стили для оформления
- `script.js` - JavaScript для интерактивности (калькулятор расчёта цены)

## Запуск

Просто откройте файл `index.html` в браузере. Для локального запуска можно использовать любой веб-сервер:

```bash
# Python 3
python -m http.server 8000

# Node.js (если установлен http-server)
npx http-server

# PHP
php -S localhost:8000
```

Затем откройте в браузере: `http://localhost:8000`

## Функциональность

- Адаптивный дизайн для всех устройств
- Интерактивный калькулятор расчёта стоимости перевозки
- Плавная прокрутка по якорным ссылкам
- Современный и чистый дизайн

### Решение проблемы с www поддоменом

**⚠️ ВАЖНО:** Не добавляйте CNAME для основного домена! Это удалит A записи от GitHub Pages.

**Правильная настройка DNS:**

1. **Основной домен (`серафим.бел` / `xn--80ajkqrey.xn--90ais`):**
   - Должен иметь **A записи** (не трогайте их!):
     - `185.199.108.153`
     - `185.199.109.153`
     - `185.199.110.153`
     - `185.199.111.153`
     - `178.124.128.83`

2. **www поддомен (`www.серафим.бел`):**
   - ❌ **УДАЛИТЕ все A записи** для www поддомена (если они есть)
   - ✅ **Добавьте CNAME запись**:
     - **Имя/Поддомен:** `www`
     - **Тип:** `CNAME`
     - **Значение/Цель:** `antonlogin0v.github.io` ⬅️ **Указывайте на GitHub Pages домен!**

**⚠️ Ошибка GitHub Pages:** "www.xn--80ajkqrey.xn--90ais is improperly configured - using A record instead of CNAME"
- Это означает, что для www поддомена настроены A записи, а нужна CNAME
- Удалите A записи для www и добавьте CNAME запись

**Почему именно так?**
- Для основного домена (apex domain) GitHub Pages требует A записи
- Для www поддомена GitHub Pages требует CNAME запись, указывающую на `antonlogin0v.github.io`
- Нельзя использовать A записи для www поддомена

### Проверка DNS записей

После добавления записи подождите несколько минут (обычно 5-30 минут) для распространения DNS изменений, затем проверьте:

```bash
# Проверка www поддомена
dig www.xn--80ajkqrey.xn--90ais CNAME

# Или используйте онлайн инструменты:
# - https://dnschecker.org
# - https://www.whatsmydns.net
```

После настройки DNS, GitHub Pages автоматически обнаружит изменения и статус станет зелёным.

### Проблема: "This connection is not private" / SSL сертификат

**Ошибка:** "Enforce HTTPS — Unavailable for your site because your domain is not properly configured to support HTTPS"

**Причина:** GitHub Pages не может выпустить SSL сертификат, потому что для www поддомена всё ещё используются **A записи** вместо **CNAME**.

**⚠️ КРИТИЧЕСКИ ВАЖНО:** Для выпуска SSL сертификата GitHub Pages требует, чтобы www поддомен использовал CNAME запись, а не A записи.

**Решение:**

1. **Удалите все A записи для www поддомена:**
   - В панели управления DNS удалите все 4 A записи для `www` поддомена
   - IP адреса, которые нужно удалить: `185.199.108.153`, `185.199.109.153`, `185.199.110.153`, `185.199.111.153`

2. **Добавьте CNAME запись для www:**
   - **Имя/Поддомен:** `www`
   - **Тип:** `CNAME`
   - **Значение:** `antonlogin0v.github.io`

3. **Проверьте настройку:**
   ```bash
   # Должна вернуться CNAME запись (не A записи!)
   dig www.xn--80ajkqrey.xn--90ais CNAME
   ```

4. **Подождите распространения DNS (5-30 минут)**

5. **После правильной настройки:**
   - GitHub Pages автоматически обнаружит изменения
   - SSL сертификат будет выпущен автоматически (обычно в течение нескольких часов)
   - Опция "Enforce HTTPS" станет доступной

**Примечание:** Пока сертификат не выпущен, сайт может быть доступен только по HTTP (небезопасно) или показывать предупреждение о безопасности. После выпуска сертификата сайт будет доступен по HTTPS без предупреждений.
